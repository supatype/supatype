# Phase 7 — Studio Dashboard

> **Definatype** · Weeks 43–48 · March 2026 · Draft

## Overview

Build the developer-facing management UI. Studio is where developers manage their database, inspect data, run queries, view migrations, manage users, and browse storage — a web-based equivalent of the CLI for visual workflows.

## Dependencies

Phase 6 complete — admin panel provides UI component patterns to reuse.

## Deliverable

Developer has a web UI for managing all aspects of their Definatype project — data, schema, auth, storage, API docs, and settings.

## Technical Context

- Studio reuses some components from the admin panel (table renderer, form widgets) but has a different layout and navigation focused on developer tools.
- The Visual Schema Designer is basic in v1 — a field list editor per model, not a full ERD drag-and-drop builder. The schema DSL in TypeScript remains the source of truth.
- API documentation is auto-generated by reading PostgREST's OpenAPI output (GET /) and rendering it with a custom UI (not Swagger UI, which is heavy and generic).
- Request logs come from Kong's logging plugin, stored in Postgres or a time-series store. In local dev, logs are minimal; in cloud, they're comprehensive.

## Risks & Mitigations

| Risk | Mitigation |
|------|------------|
| Studio scope — every developer tool could be its own product | Ship functional-but-basic versions of each tool; iterate based on user feedback |
| Data Explorer performance with large tables | Server-side pagination only; limit inline editing to single records; lazy-load JSON fields |
| SQL query runner security — user could break the database | Run queries as a restricted Postgres role; add read-only mode option; warn before DDL statements |

## Success Criteria

Phase 7 is complete when:

- [ ] Studio loads and displays all project models in the schema designer
- [ ] Data Explorer browses records with pagination and filtering
- [ ] SQL query runner executes queries and displays results
- [ ] Migration history shows all applied migrations with SQL detail
- [ ] Auth management lists users and allows role changes
- [ ] Storage browser displays buckets and files with upload/download
- [ ] API documentation renders all available endpoints
